ESP32 8-Channel Relay: Technical Deep Dive

This project showcases a sophisticated implementation of embedded systems and Human-Computer Interaction (HCI). By scaling from a standard 4-channel to an 8-channel system, this hub provides a robust solution for complex automation tasks controlled via a custom web application.

Detailed Hardware Pinout

Relay Channel ESP32 GPIO Description
Relay 1 & 2 GPIO 13, 12 Primary Control Channels
Relay 3 & 4 GPIO 14, 27 Auxiliary Control Channels
Relay 5 & 6 GPIO 26, 25 Secondary Expansion Pins
Relay 7 & 8 GPIO 33, 32 High-Index Control Pins
JD-VCC 12V Ext. External Coil Supply

Operational Protocol:

  • Access: Connect to the local "ESP32-8Channel-WiFi" network.
  • Control: Launch the web app and initialize the Bluetooth handshake for low-latency commands.
  • Redundancy: If Bluetooth becomes unavailable, the system utilizes standard HTTP fetch requests over the local Wi-Fi gateway.

Safety Architecture Note:

Removing the VCC/JD-VCC jumper is critical. This ensures galvanic isolation, preventing electromagnetic interference from the 12V relay coils from affecting the ESP32’s operation.
